================================================================================
                        1. ARQUITECTURA DEL PROYECTO
================================================================================

El proyecto implementa una arquitectura cliente-servidor moderna con:

BACKEND (Spring Boot 3.3.0)
---------------------------
- Framework: Spring Boot 3.3.0 (Spring Framework 6.x)
- Lenguaje: Java 22
- Gestor de Dependencias: Maven
- Base de Datos: PostgreSQL 13+
- Arquitectura: API REST con patrón MVC
- Puerto de Ejecución: 8080

FRONTEND (Angular 17)
---------------------
- Framework: Angular 17.3.0
- Lenguaje: TypeScript 5.4.2
- Estilos: SCSS + Tailwind CSS 3.4.17
- Arquitectura: SPA (Single Page Application)
- SSR: Soporte para Server-Side Rendering
- Puerto de Ejecución: 4200

================================================================================
                    2. MÓDULOS IMPLEMENTADOS - BACKEND
================================================================================

2.1 AUTENTICACIÓN Y SEGURIDAD (auth/)
--------------------------------------
✓ Sistema de autenticación JWT (JSON Web Tokens)
✓ Registro de usuarios con validación
✓ Login con generación de tokens
✓ Sistema de roles (ROLE_ADMIN, ROLE_USER, ROLE_SELLER)
✓ Filtro de autenticación para proteger endpoints
✓ Encriptación de contraseñas con BCrypt

Endpoints Implementados:
- POST /api/auth/register - Registro de usuarios
- POST /api/auth/login - Inicio de sesión
- Autenticación automática en cada request

Entidades:
- User: Información del usuario
- Role: Roles del sistema
- AuthRequest/AuthResponse: DTOs para auth
- RegisterRequest: DTO para registro

2.2 GESTIÓN DE PRODUCTOS (product/)
------------------------------------
✓ CRUD completo de productos
✓ Paginación y ordenamiento
✓ Filtrado por categorías
✓ Búsqueda de productos
✓ Gestión de stock e inventario
✓ Precios y comparación de precios
✓ Productos destacados (featured)
✓ Atributos ecológicos vinculados

Endpoints Implementados:
- GET /api/products - Listar productos con paginación
- GET /api/products/{id} - Obtener producto específico
- POST /api/products - Crear producto (Admin)
- PUT /api/products/{id} - Actualizar producto (Admin)
- DELETE /api/products/{id} - Eliminar producto (Admin)
- GET /api/products/featured - Productos destacados
- GET /api/products/search - Búsqueda avanzada

Entidades:
- Product: Información del producto
- ProductRequest/ProductResponse: DTOs
- PagedProductResponse: Respuesta paginada

Características Especiales:
- Validación de stock
- Control de productos activos/inactivos
- SKU único por producto
- Soporte para múltiples imágenes

2.3 GESTIÓN DE CATEGORÍAS (category/)
--------------------------------------
✓ CRUD de categorías
✓ Jerarquía de categorías (parent-child)
✓ Slug único para URLs amigables
✓ Ordenamiento personalizado
✓ Activación/desactivación de categorías

Endpoints Implementados:
- GET /api/categories - Listar todas las categorías
- GET /api/categories/{id} - Obtener categoría específica
- POST /api/categories - Crear categoría (Admin)
- PUT /api/categories/{id} - Actualizar categoría (Admin)
- DELETE /api/categories/{id} - Eliminar categoría (Admin)

Entidades:
- Category: Información de categoría
- CategoryRequest/CategoryResponse: DTOs

2.4 GESTIÓN DE ÓRDENES (order/)
--------------------------------
✓ Creación de órdenes de compra
✓ Estados de orden (pending, confirmed, processing, shipped, delivered, cancelled)
✓ Estados de pago (pending, paid, failed, refunded)
✓ Estados de envío (not_shipped, preparing, shipped, delivered)
✓ Cálculo automático de totales
✓ Vinculación con direcciones de envío
✓ Gestión de items de orden

Endpoints Implementados:
- GET /api/orders - Listar órdenes del usuario
- GET /api/orders/{id} - Obtener orden específica
- POST /api/orders - Crear nueva orden
- PUT /api/orders/{id}/status - Actualizar estado (Admin)
- GET /api/orders/all - Listar todas las órdenes (Admin)

Entidades:
- Order: Información de la orden
- OrderItem: Items individuales de la orden
- OrderStatus: Enum de estados
- OrderRequest/OrderResponse: DTOs
- OrderItemRequest/OrderItemResponse: DTOs

Características Especiales:
- Generación automática de número de orden (formato: ECO-YYYYMMDD-XXXXXX)
- Cálculo de subtotales, impuestos y envío
- Descuentos aplicables
- Snapshot de precios al momento de la compra

2.5 GESTIÓN DE IMÁGENES (images/)
----------------------------------
✓ Carga de imágenes de productos
✓ Almacenamiento local en servidor
✓ Servicio de recuperación de imágenes
✓ Soporte para múltiples formatos

Endpoints Implementados:
- POST /api/images/upload - Subir imagen
- GET /api/images/{filename} - Obtener imagen

================================================================================
                        13. CÓMO EJECUTAR EL PROYECTO
================================================================================

BACKEND:
--------
1. Configurar PostgreSQL (crear database 'ecomarket')
2. Actualizar application-dev.properties con credenciales
3. Ejecutar: cd backend && mvn clean install
4. Ejecutar: mvn spring-boot:run
5. API disponible en: http://localhost:8080

FRONTEND:
---------
1. Instalar dependencias: cd frontend-angular && npm install
2. Ejecutar: npm start
3. App disponible en: http://localhost:4200

BASE DE DATOS:
--------------
1. Crear database: CREATE DATABASE ecomarket;
2. Ejecutar script: psql -d ecomarket -f database_schema.sql
3. El schema incluye datos iniciales automáticamente

DOCKER:
-------
1. cd backend
2. docker build -t ecomarket-backend .
3. docker run -p 8080:8080 ecomarket-backend

El proyecto EcoMarket es una plataforma e-commerce completa y funcional que 
cumple con todos los requerimientos principales de un sistema de comercio 
electrónico moderno. Destaca por:

1. ARQUITECTURA ROBUSTA: Implementación profesional con Spring Boot 3 y 
   Angular 17, siguiendo las mejores prácticas de desarrollo.

2. FUNCIONALIDAD COMPLETA: Desde el registro de usuarios hasta la gestión

3. ENFOQUE ECOLÓGICO: Sistema especializado con atributos ambientales y de 
   sostenibilidad para cada producto.
